Реализованные принципы SOLID:

Принцип единственной ответственности:
В качестве реализации приведен класс GameController (...\Controllers\GameController.cs), который отвечает за конвертацию данных и передачи результата конвертации в верхний слой.

Принцип инверсии зависимостей:
	В качестве реализации приведен класс GameInteractor (...\GameInteractor.cs), который принимает в качестве зависимостей абстракции, а не детали, тем самым, верхний слой не зависит от деталей.

Принцип разделения интерфейса:
	В качестве реализации приведены два интерфейса IGameController и IConfigController. IGameController отвечает за обработку запросов для игровой логики, а IConfigController для конфигурационной.

Принцип открытости/закрытости:
	В качестве реализации приведен абстрактный класс ConfigController (...\Abstractions\ConfigController.cs), который можно расширять различными реализациями.

Принцип подстановки Барбары Лисков:
	В качестве реализации приведены классы ConsoleConfigController и FileConfigController, которые не усиливают предусловия и не ослабляют постусловия базового класса.
